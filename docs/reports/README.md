# Phase 1 Reports

## Weekly Snapshot Routine
1. Freitag: Exportiere Board-Status (Tickets, Velocity, Blocker-Liste).
2. Speichere CSV/Markdown unter `docs/reports/YYYY-MM-DD-weekly.md`.
3. Notiere Highlights (Done, In Progress, Blocked) + KPIs (alerts latency, QA pass rate, signups).
4. Verlinke relevante PRs, Figma Links, Monitoring Dashboards.
5. Aktualisiere `changelog-phase1.md` mit kurzem Hinweis auf den Report.

## Optional Add-ons
- Burndown Chart Screenshots (`reports/assets/`).
- UX Preview Links.
- Data Quality summary vom Data Engineering Droid.

Halte die Reports konsistent, damit Management & Droids den Fortschritt auf einen Blick sehen.
